# Запуск симулятора на сайті sim.docaa.net
## Покрокова інструкція — без технічного досвіду

---

## Що таке що (щоб розуміти про що мова)

**VPS** — це орендований комп'ютер в інтернеті. На ньому буде жити симулятор.
**Terminal / SSH** — програма щоб керувати тим комп'ютером через текстові команди.
**Node.js** — програма яка запускає наш симулятор.
**База даних** — файл де зберігаються всі студенти, групи, прогрес.
**Nginx** — програма яка відкриває доступ до симулятора через браузер.

---

## ЧАСТИНА 1 — Підготовка (робиться один раз)

---

### Крок 1. Орендуйте сервер

Потрібен VPS — орендований комп'ютер в інтернеті.

**Де орендувати (вибрати один):**
- **Hetzner** (hetzner.com) — найдешевше, €4-6/міс, надійно
- **DigitalOcean** (digitalocean.com) — $6/міс
- **Hostinger** (hostinger.com) — є українська підтримка

**Що обирати при замовленні:**
```
Операційна система: Ubuntu 22.04 LTS
RAM: мінімум 2 GB
Диск: мінімум 20 GB
```

Після оплати вам дадуть:
- **IP адресу** — наприклад `95.216.45.123`
- **пароль root** або SSH ключ

Запишіть IP адресу — вона потрібна в наступних кроках.

---

### Крок 2. Налаштуйте піддомен sim.docaa.net

Це щоб симулятор відкривався за адресою `sim.docaa.net`.

1. Зайдіть на **imena.ua** (де у вас домен docaa.net)
2. Знайдіть розділ **DNS** або **Управління DNS**
3. Додайте новий запис:

```
Тип записи:  A
Ім'я (Name): sim
Значення:    95.216.45.123  ← ваша IP адреса сервера
TTL:         3600
```

4. Збережіть.

**Як перевірити що спрацювало** (через 15-30 хвилин):
Зайдіть на https://dnschecker.org і введіть `sim.docaa.net`
Має показати вашу IP адресу.

---

### Крок 3. Підключіться до сервера

Це як відкрити папку на тому орендованому комп'ютері через програму.

**На Windows** — завантажте програму **PuTTY**: https://putty.org
- Host Name: `95.216.45.123` (ваша IP)
- Port: `22`
- Натисніть Open
- Введіть логін: `root`
- Введіть пароль (той що дав хостинг)

**На Mac** — відкрийте Terminal (Spotlight → Terminal):
```
ssh root@95.216.45.123
```
Введіть пароль.

Ви всередині сервера — тепер вводите команди і вони виконуються на тому комп'ютері.

---

### Крок 4. Встановіть Node.js

Node.js — це програма яка запускає симулятор.

Скопіюйте і вставте ці команди **по одній**, чекайте поки кожна виконається:

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
```
*(чекати ~30 секунд)*

```bash
sudo apt-get install -y nodejs
```
*(чекати ~1 хвилину)*

```bash
node --version
```
Має показати: `v20.x.x` — якщо показує, все добре.

---

### Крок 5. Встановіть допоміжні програми

```bash
sudo apt update
```
*(чекати ~30 секунд)*

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```
*(чекати ~2 хвилини)*

```bash
npm install -g pm2
```
*(чекати ~30 секунд)*

**Nginx** — відкриває доступ через браузер
**PM2** — стежить щоб симулятор завжди працював (навіть після перезавантаження)

---

### Крок 6. Завантажте файли симулятора на сервер

Вам потрібна програма **FileZilla** (безкоштовна): https://filezilla-project.org

1. Відкрийте FileZilla
2. Вгорі введіть:
   - Хост: `sftp://95.216.45.123`
   - Ім'я користувача: `root`
   - Пароль: (ваш пароль від сервера)
   - Порт: `22`
3. Натисніть **Швидке з'єднання**
4. Ліворуч — ваш комп'ютер. Праворуч — сервер.
5. На сервері (праворуч) перейдіть в папку `/var/www/`
6. Правою кнопкою → **Створити папку** → назвіть `sim-docaa`
7. Зайдіть в папку `sim-docaa`
8. Розпакуйте архів `sim-server.zip` на своєму комп'ютері
9. Перетягніть **вміст** папки `sim-server` у FileZilla праворуч

Структура на сервері має бути:
```
/var/www/sim-docaa/
    package.json
    .env.example
    server/
    scripts/
    public/
    README.md
```

---

### Крок 7. Налаштуйте секретні параметри (.env файл)

Поверніться в Terminal/PuTTY.

```bash
cd /var/www/sim-docaa
cp .env.example .env
nano .env
```

Відкриється редактор. Стрілками переміщуйтесь і заповніть:

```
PORT=3000
NODE_ENV=production
JWT_SECRET=     ← тут потрібен довгий випадковий рядок (дивіться нижче)
ANTHROPIC_API_KEY=sk-ant-api03-...   ← ваш ключ
DB_PATH=./data/simulator.db
FRONTEND_URL=https://sim.docaa.net
```

**Як згенерувати JWT_SECRET** (новий рядок в терміналі):
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```
Скопіюйте результат (довгий рядок букв і цифр) і вставте після `JWT_SECRET=`

**Зберегти файл:** натисніть `Ctrl+O` → Enter → `Ctrl+X`

---

### Крок 8. Встановіть залежності (бібліотеки)

```bash
cd /var/www/sim-docaa
npm install
```
*(чекати ~1 хвилину — завантажує бібліотеки)*

## ЧАСТИНА 2 — База даних

---

### Крок 9. Створіть базу даних

База даних — це файл де будуть зберігатися:
- всі студенти і лектори
- групи
- прогрес кожного студента
- переписки, замовлення

```bash
node scripts/init-db.js
```

Має показати:
```
✅ Database initialized: ./data/simulator.db
```

**Що відбулось:** створився файл `data/simulator.db` — це і є база даних.

---

### Крок 10. Створіть першого адміністратора (ви)

```bash
node scripts/create-admin.js
```

Програма запитає:
```
Email:    ← введіть ваш email
Name:     ← введіть ваше ім'я
Password: ← придумайте пароль (мінімум 8 символів)
```

Має показати:
```
✅ Superadmin created: your@email.com
```

**Запишіть цей email і пароль** — він потрібен для входу як адміністратор.

---

### Крок 11. Скопіюйте файл симулятора

```bash
cp /var/www/sim-docaa/public/freightdesk-v4.html /var/www/sim-docaa/public/simulator.html
```

*(якщо файл freightdesk-v4.html є в папці public)*

## ЧАСТИНА 3 — Запуск

---

### Крок 12. Запустіть симулятор

```bash
cd /var/www/sim-docaa
pm2 start server/index.js --name "docaa-sim"
```

Має показати таблицю з рядком де написано `online`.

```bash
pm2 save
```

```bash
pm2 startup
```
Ця команда покаже ще одну команду — скопіюйте і виконайте її. Це потрібно щоб симулятор запускався автоматично після перезавантаження сервера.

**Перевірте що працює:**
```bash
curl http://localhost:3000/health
```
Має показати: `{"ok":true,...}`

---

### Крок 13. Налаштуйте веб-доступ (Nginx)

```bash
nano /etc/nginx/sites-available/sim.docaa.net
```

Вставте наступний текст (Ctrl+Shift+V або правою кнопкою в PuTTY):

```
server {
    listen 80;
    server_name sim.docaa.net;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Зберегти: `Ctrl+O` → Enter → `Ctrl+X`

```bash
ln -s /etc/nginx/sites-available/sim.docaa.net /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

---

### Крок 14. Увімкніть HTTPS (замок в браузері)

```bash
certbot --nginx -d sim.docaa.net
```

Програма задасть питання:
- Введіть email (для повідомлень про сертифікат)
- Погодьтесь з умовами: `A`
- На питання про перенаправлення оберіть: `2` (Redirect)

---

### Крок 15. Перевірте що все працює

Відкрийте браузер і зайдіть на:
```
https://sim.docaa.net
```

Має відкритись сторінка входу Docaa.

Введіть email і пароль адміністратора з Кроку 10.

---

## ЧАСТИНА 4 — Перший запуск (після деплою)

---

### Крок 16. Структура роботи

```
ВИ (суперадмін)
    ↓ створюєте лектора
ЛЕКТОР
    ↓ створює групу, додає студентів
СТУДЕНТ
    ↓ логіниться, проходить симуляцію
```

### Крок 17. Як створити лектора

1. Зайдіть на `https://sim.docaa.net` під адміністратором
2. Відкриється адмін-панель
3. Кнопка "Новий лектор"
4. Введіть ім'я, email, пароль
5. Лектор отримує лист або ви передаєте йому дані входу

### Крок 18. Що робить лектор

1. Логіниться на `https://sim.docaa.net` — відкривається кабінет лектора
2. Створює групу (наприклад "Логістика 2026 — Група 1")
3. Додає студентів — вводить ім'я, email, пароль
4. Система автоматично генерує кожному студенту набір листів
5. Лектор може: переглядати прогрес, зупиняти/відновлювати сесію, змінювати набір листів

### Крок 19. Що бачить студент

1. Логіниться на `https://sim.docaa.net`
2. Одразу відкривається симулятор — без жодних зайвих кнопок
3. Прогрес зберігається автоматично кожні 30 секунд
4. Якщо закрив браузер — відкриває знову і продовжує з того місця

## Часті питання

**Симулятор завис або не відповідає:**
```bash
pm2 restart docaa-sim
```

**Подивитись що відбувається (логи):**
```bash
pm2 logs docaa-sim --lines 50
```

**Зробити резервну копію бази даних:**
```bash
cp /var/www/sim-docaa/data/simulator.db ~/backup-$(date +%Y%m%d).db
```

**Завантажити нову версію симулятора:**
Завантажте новий файл через FileZilla в папку `public` під назвою `simulator.html`
Перезапуск не потрібен.

**Скільки коштує API (Anthropic):**
Орієнтовно $0.10–$0.30 на одного студента за повну сесію (всі 8 замовлень + переговори).
500 студентів ≈ $50–$150.
