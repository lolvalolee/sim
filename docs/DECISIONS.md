### [раніше] Детермінований торг B
**Контекст:** AI пробивав межі торгу (3500→1900, +700 за «послуги»).
**Рішення:** Ціну рахує код (`carrierNextOffer`, `clientNextOffer`) у `simulator.html`; AI отримує директиву «назви EUR X». Стан у БД (`migrate-v24`: nego_offer, nego_ref).
**Наслідки:** Межі ref−300..ref+100 (перевізник), ref..ref+300 (замовник), макс комісія 600.

### [раніше] Варіант 2 — закріплення рейсу за чатом
**Контекст:** Один чат плутав два рейси; усі отримували однаковий ref.
**Рішення:** `carrier_chats.locked_letter_id` (`migrate-v25`); `matchOrderByRoute` визначає рейс за містами з повідомлення студента; PATCH `/chats/:id/lock`.
**Наслідки:** Один чат = один рейс = один торг. Закріплення ≠ угода (`session_deals`).

### [раніше] Підтвердження угоди — кнопка, не AI
**Контекст:** AI-verdict давав непостійність (кнопка світилась/гасла).
**Рішення:** Кнопка «Підтвердити» завжди фіксує угоду; `session_deals` — джерело істини.
**Наслідки:** Стабільний deal-bar; AI лише для тексту відповіді.

### [раніше] Інциденти за сим-годиною
**Контекст:** Усі етапи дня вистрілювали за секунди.
**Рішення:** `target_sim_hour_abs` + cron звіряє абсолютну сим-годину.
**Наслідки:** Етапи рознесені; при reset — wipe інцидентів.

### [раніше] Один файл фронту
**Контекст:** Історично `freightdesk-v4.html` → `simulator.html`.
**Рішення:** Весь UI+JS в одному HTML (~568KB). Без bundler.
**Наслідки:** Простий деплой; важко навігувати — шукати за функціями/grep.

### [2025-06-24] Система пам'яті проєкту
**Контекст:** Cursor не бачить PDF, транскрипти, історію рішень.
**Рішення:** П'ять файлів у `docs/`: `AI_CONTEXT.md`, `DECISIONS.md`, `ROADMAP.md`, `BUSINESS_RULES.md`, `CHANGELOG.md`. AI читає їх перед задачами.
**Наслідки:** Єдине джерело контексту для агентів; handoff у `docs/handoff/` — архів.

### [2025-06-24] Папка docs/ — все не-серверне в одній папці
**Контекст:** Власник пушить увесь репо через Sublime Merge; на VPS потрібен лише runtime.
**Рішення:** У корені — код; документація в `docs/`. `deploy.sh` виключає `docs/`.
**Наслідки:** Зручно комітити все в git; на сервер не потрапляє зайве.

### [2025-06-24] Біржа: обов'язковий вибір рейсу + жорсткий фільтр напрямку
**Контекст:** `dirsNeeded=[]` пропускав фільтр напрямку → UA-FI перевізник на DE→UA; `letter_id` не зберігався на біржі.
**Рішення:**
1. Форма біржі: dropdown рейсу (обов'язковий), автозаповнення маршруту/ТЗ
2. `cargo_board.letter_id` (`migrate-v26`)
3. Прибрано `dirsNeeded.length === 0 ||` у фільтрі — без напрямку 0 відгуків
4. `server/utils/route-matcher.js` — спільний матчинг міст (кирилиця/латиниця, аліаси)
5. Покращено `matchOrderByRoute` на фронті (ті самі аліаси)
**Наслідки:** `freight_ref` і `dirs` завжди точні при пості з листа; fallback-матчинг для ручного маршруту.
