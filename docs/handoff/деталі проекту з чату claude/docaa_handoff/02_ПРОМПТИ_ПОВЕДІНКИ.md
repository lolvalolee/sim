# ПРОМПТИ ПОВЕДІНКИ AI — ЗАМОВНИК І ПЕРЕВІЗНИК

Це найважливіший файл для AI-поведінки. Усі системні промпти будуються
НА КЛІЄНТІ (у simulator.html), бо AI викликається з браузера через проксі
`/api/student/ai`. Дві головні функції: buildClientSys (замовник) і
buildCarrierSys (перевізник).

ВАЖЛИВО ПРО АРХІТЕКТУРУ:
- Промпт = системне повідомлення. Історія чату передається окремо.
- Перед викликом AI код додає СУВОРУ ДИРЕКТИВУ торгу (negoDirective) з готовою
  цифрою — AI лише формулює фразу навколо неї, число НЕ вигадує.
- Жорсткі дані (маршрут, км, адреси, ціна-орієнтир) — з letters_v2/letters, AI не змінює.
- М'які дані (тип ТЗ-альтернативи) — можна обговорювати.
- Приховані дані — AI знає, але мовчить поки студент не запитає (частина завдання).

## ЛОГІКА ВИЗНАЧЕННЯ РЕЙСУ (де береться контекст для промпта)

Перед buildCarrierSys код визначає ordCtx (про який рейс мова) — Варіант 2:
1. Якщо чат закріплений (S.chatLockedLetter[cid]) → беремо той рейс.
2. Якщо перевізник має угоду (carrierId) → той рейс.
3. Інакше → matchOrderByRoute(текст_студента) шукає рейс за містами → закріплює.
4. Якщо не визначено → перевізник ПИТАЄ маршрут, ціну не називає.

freightRef рейсу = letters_v2.freight_ref. Якщо ordCtx=null → fref=0 →
ТОРГ ВИМИКАЄТЬСЯ (це і є поточний баг, див. 06).

## КЛЮЧОВІ ПРАВИЛА В ПРОМПТАХ (підсумок)

### Замовник (buildClientSys):
- Орієнтир бюджету = freight_ref. НЕ називає його прямо.
- Платить: до ref — погоджується; ref..ref+300 — трохи збиває ("є варіанти кращі",
  −100/крок); вище ref+300 — "дорого, можемо ref?"; СТЕЛЯ ref+300, далі "Остаточно ні".
- Підозріло дешеве (<60% ref) — рандомно: насторожується АБО погоджується.
- Усі ціни кратні 50. Ввічливість НЕ змінює стелю.
- Правило ваги: не плутати палети з тоннами; стеля тент 24т / реф 22т / негабарит — з даних.
- Приховані дані — розкриває лише на прямий запит (складність = відсутність даних).
- Формат адрес за країною призначення.
- Якщо угоду закрито — лише обмін деталями, НЕ новий торг.

### Перевізник (buildCarrierSys):
- Характер з PERSONA_PROMPTS (tough/local/eastern/unreliable/polish/fridge/adr/specialist/agrarian).
- Мова за національністю (PL→польською, OTHER→англ, інакше укр).
- Якщо тип ТЗ не його — відмовляє ("у нас немає рефів, тільки тенти").
- Обговорює ТІЛЬКИ той вантаж що в контексті. Не вигадує інший маршрут.
- Унікальні дані водія/авто/тел з seedN (не шаблонні).
- Дата завантаження: якщо НЕ вказана — питає, НЕ вигадує.
- Торг: цифру дає КОД через negoDirective. Перевізник лише озвучує.

### СУВОРА ДИРЕКТИВА ТОРГУ (negoDirective, додається в історію перед викликом AI):
- action 'agree': "погодься на EUR X, назви саме цю цифру і підтверди"
- action 'hold': "твоя ОСТАТОЧНА EUR X, нижче не підеш, назви саме її"
- action 'concede': "трохи поступись — назви EUR X (не нижче), мінімум EUR floor"
- Якщо рейс не визначено: "не знаєш про який рейс — запитай маршрут, ціну НЕ називай"
- Якщо інший рейс закріпленому: "авто зайняте під поточну домовленість"
