# DOCAA FREIGHT SIMULATOR — ХЕНДОФ ДЛЯ CURSOR

Це навчальний тренажер експедитора (українськомовний, веб). Студент грає роль
експедитора: отримує листи від замовників, домовляється з ними про ціну,
шукає перевізників, торгується, створює заявки, веде рейс через інциденти,
готує документи. AI грає замовників і перевізників.

## СТЕК
- Node.js / Express, better-sqlite3 (СИНХРОННИЙ — ніколи .catch на запитах)
- SQLite, Ubuntu VPS `/var/www/sim-docaa/`, домен sim.docaa.net
- PM2 процес `docaa-sim`
- Фронт = ОДИН файл `public/simulator.html` (~568 KB, ~10600 рядків, весь UI+логіка)
- AI: Claude (claude-sonnet-4-5) через серверний проксі `/api/student/ai` (один ключ на систему)
- Репо: lolvalolee/sim (раніше деплой через Sublime Merge, тепер переходимо в Cursor)

## ДИЗАЙН
Темна тема. Фон #151515, акцент #FFBE63, шрифт Inter, лого SVG. Viber-стиль чатів.

## КЛЮЧОВІ ФАЙЛИ
- `public/simulator.html` — фронт (UI, увесь клієнтський JS, промпти AI будуються ТУТ)
- `server/routes/student.js` — основні API студента (чати, заявки, торг-persist, документи)
- `server/utils/incident-scheduler.js` — інциденти рейсу за сим-часом
- `server/routes/lecturer.js`, `public/lecturer.html` — кабінет лектора
- `server/routes/admin.js`, `public/superadmin.html` — суперадмін
- `scripts/migrate-vN.js` — міграції БД (остання застосована: v25)
- `scripts/seed-*.js` — заповнення даних (letters, carriers, clients, letters_v2)

## ТРИ РОЛІ
superadmin (створює лекторів) → lecturer (групи, запуск сесій) → student (грає симуляцію)

## ДЖЕРЕЛА ДАНИХ (Excel/ODS, прив'язані до рейсів)
- `Симулятор_1_1.ods` / `1_2` — маршрути, фрахт, відстані (км), прикордонні переходи, інтервали часу
- `Симулятор_2_0` → таблиця `letters_v2` (120 рейсів): freight_ref (орієнтир €), rate_basis,
  data_required, hidden_task, vehicle_required/alternative, total_km, km_before/after_border,
  direction (імпорт/експорт), incoterms, difficulty_v2, border_short
- letters (120) ↔ letters_v2 (120) зв'язані по letter_id. НА ПРОДІ: 120/120/120 linked (цілий).

## ГОЛОВНІ ПРИНЦИПИ ВЛАСНИКА (Luke)
1. Детермінований код > промпт. AI не тримає межі/факти — цифри й рішення рахує КОД,
   AI лише формулює фразу. Це доведено багато разів (торг пробивався, поки рахував AI).
2. Обговорити корінь ПЕРЕД кодом для складних фіч. Не латати симптом.
3. Не ламати робоче. Точкові правки, не великі рефактори.
4. Не дробити надмірно, не over-engineer ("ти перемудрував").
5. AI замовника/перевізника має ПАМ'ЯТАТИ все: оригінальний лист, переписку, вантаж, домовленості.
6. Усі дані (км, ціна) — з файлу Симулятор, прив'язані до рейсу. AI їх НЕ переписує.
